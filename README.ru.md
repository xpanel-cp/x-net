## Введение

**X-NET** — панель для создания и управления сервисами **VPN/Proxy** и **SSH-аккаунтами**. С её помощью вы создаёте пользователей (подписки), контролируете трафик и срок действия каждого из них, а также выдаёте ссылки на подписку.

- Обработка трафика на ядре **Sing-box**
- Возможность создания **SSH-аккаунтов** с различными протоколами: SSH-over-WebSocket, Stunnel/TLS, SlowDNS, Dropbear и BadVPN/UDPGW
- **Изоляция доступа:** каждый SSH-аккаунт имеет доступ только к своему порту/протоколу
- Управление **несколькими серверами** из единого интерфейса: один сервер «панель» и остальные «узлы»

## Установка

### Требования

- Linux-сервер (рекомендуется Ubuntu / Debian), архитектура `amd64`
- Доступ с правами `root`

### Установка панели

```bash
apt update
bash <(curl -fsSL https://raw.githubusercontent.com/xpanel-cp/x-net/main/install/xnet.sh)
```

Установщик отображает 3 последние версии в меню. Для прямой установки конкретной версии:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/xpanel-cp/x-net/main/install/xnet.sh) v1.2.3
```

При первоначальной установке запрашивается порт панели и данные администратора, создаётся служба systemd, устанавливается ядро sing-box и SSH-подсистемы, а порт открывается в брандмауэре. При повторном запуске (обновлении) файл `.env` и база данных сохраняются.

### Установка узла (Agent)

Запустите ту же команду установки на сервере узла и выберите `agent` в качестве роли при запросе. Затем в панели перейдите в **Серверы и узлы → Зарегистрировать узел**; поместите сгенерированные ключи в `/opt/xnet/.env` на узле и перезапустите службу:

```bash
nano /opt/xnet/.env
#   NODE_ROLE=agent
#   NODE_API_KEY=xnetnode_...
#   NODE_SECRET_KEY=...
systemctl restart xnet
```

### Управление службой

```bash
systemctl status xnet      # Статус службы
journalctl -u xnet -f      # Живые логи
systemctl restart xnet     # Перезапуск службы
```

> **Безопасность:** Адрес `http://IP:PORT` не зашифрован. В производственной среде разместите панель за доменом + HTTPS.

## Возможности

### Управление входящими подключениями
Создание, редактирование, клонирование, включение/отключение и развёртывание на нескольких узлах

### Управление подписками
Ссылка на подписку, QR-код, лимит трафика, дата истечения, лимит одновременных устройств, продление и сброс трафика

### SSH-аккаунты
Создание системных пользователей, ограничение трафика и количества одновременных входов

### Многоузловость
Роли panel/agent, автоматическая синхронизация и **единый активный узел (Follow-Me)** с переключением узла из ссылки на подписку

### Безопасность
Вход через JWT, двухфакторная аутентификация (TOTP), роли доступа (Администратор/Оператор/Реселлер) и подпись HMAC между панелью и узлом

### Домен и сертификат
Реальная проверка DNS, выпуск/обновление TLS-сертификата (Let's Encrypt)

### Маршрутизация и сеть
Правила маршрутизации (GeoSite/GeoIP, AdBlock), настройка DNS и управление перегрузкой TCP BBR

### Мониторинг
Дашборд, анализ трафика, живые логи, резервное копирование/восстановление базы данных

### API
Bearer-токен, документация и playground, брандмауэр с белым списком IP-адресов

### Интерфейс
Многоязычный (персидский/английский/русский/китайский), светлая и тёмная темы, автоматическое обновление страниц
