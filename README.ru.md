## Введение
Проект **X-NET** — это панель для создания и управления сервисами **VPN/Proxy** и **аккаунтами SSH**. Через неё вы создаёте пользователей (подписки), контролируете их трафик и срок действия, а также предоставляете ссылки на подписку.

- Обработка трафика с помощью ядра **Sing-box**
- Возможность создания **SSH-аккаунтов** с различными протоколами: SSH-over-WebSocket, Stunnel/TLS, SlowDNS, Dropbear и BadVPN/UDPGW
- **Изоляция доступа:** Каждый SSH-аккаунт имеет доступ только к своему порту/протоколу
- Управление **несколькими серверами** из единого интерфейса: один сервер «панель» и остальные «ноды»

## Установка
### Предварительные требования
- Linux-сервер (рекомендуется Ubuntu / Debian), архитектура `amd64`
- Доступ `root`

### Установка панели
```bash
apt update
bash <(curl -fsSL https://raw.githubusercontent.com/xpanel-cp/x-net/main/install/xnet.sh)
```

Установщик показывает последние 3 версии в меню; для прямой установки конкретной версии:
```bash
bash <(curl -fsSL https://raw.githubusercontent.com/xpanel-cp/x-net/main/install/xnet.sh) v1.2.3
```

При первой установке запрашиваются порт панели и данные администратора, создаётся systemd-сервис, устанавливается ядро sing-box и подсистемы SSH, порт открывается в фаерволе. При повторном запуске (обновление) файл `.env` и база данных сохраняются.

### Установка ноды (Agent)
Выполните ту же команду установки на сервере ноды и выберите роль `agent`. Затем в панели перейдите в **Серверы и ноды → Register Node**; вставьте сгенерированные ключи в `/opt/xnet/.env` ноды и перезапустите сервис:
```bash
nano /opt/xnet/.env
# NODE_ROLE=agent
# NODE_API_KEY=xnetnode_...
# NODE_SECRET_KEY=...
systemctl restart xnet
```

### Управление сервисом
```bash
systemctl status xnet # Статус сервиса
journalctl -u xnet -f # Живые логи
systemctl restart xnet # Перезапуск
```

> **Безопасность:** Адрес `http://IP:PORT` не зашифрован; для продакшена разместите панель за доменом + HTTPS.

## Возможности
### Управление Inbound
Создание, редактирование, клонирование, включение/отключение и развертывание на нескольких нодах

### Управление подписками
Ссылки на подписку, QR-код, лимит объёма, дата истечения, лимит одновременных устройств, продление и сброс трафика

### Аккаунты SSH
Создание системных пользователей, ограничение трафика и количества одновременных входов

### Многоузловая архитектура
Роли panel/agent, автоматическая синхронизация и **одна активная нода (Follow-Me)** с переключением ноды через ссылку подписки

### Безопасность
Вход через JWT, двухфакторная аутентификация (TOTP), роли доступа (админ/оператор/представитель) и HMAC-подпись между панелью и нодами

### Домен и сертификаты
Проверка реального DNS, выдача/продление TLS-сертификатов (Let's Encrypt)

### Маршрутизация и сеть
Правила Routing (GeoSite/GeoIP, AdBlock), настройка DNS и контроль перегрузки TCP BBR

### Мониторинг
Дашборд, анализ трафика, живые логи, резервное копирование/восстановление базы данных

### API
Bearer-токен, документация и playground, whitelist IP в фаерволе

### Пользовательский интерфейс
Мультиязычный (персидский/английский/русский/китайский), светлая и тёмная темы, автоматическое обновление страниц
