## معرفی پروژه

**X-NET** یک کنترل‌پنل کامل و مدرن برای مدیریت سرویس‌های VPN/Proxy و تونل‌های SSH است. هستهٔ پردازش ترافیک بر پایهٔ **Sing-box** بنا شده و بک‌اند با زبان **Go** برای کارایی بالا و مصرف کم حافظه توسعه یافته است. فرانت‌اند با **React + Vite** ساخته شده و رابطی چندزبانه (فارسی، انگلیسی، روسی، چینی) با تم روشن/تاریک ارائه می‌دهد.

این پنل برای مدیریت **چند سرور (multi-node)** طراحی شده است: یک سرور نقش **پنل (control-plane)** را دارد — صاحب دیتابیس، رابط مدیریت و هماهنگ‌کننده — و سرورهای دیگر به‌عنوان **نود (agent / data-plane)** اینباندها و اکانت‌ها را اجرا می‌کنند. همه‌چیز از یک رابط واحد مدیریت می‌شود.

### چرا X-NET؟

- معماری **تک‌پورت**: فرانت، بک‌اند و WebSocket روی یک پورت سرو می‌شوند.
- نصب بدون نیاز به Go یا Node روی سرور (نصب از باینری از‌پیش‌کامپایل‌شده).
- دیتابیس بومی **SQLite (WAL)** — بدون وابستگی به سرویس دیتابیس خارجی.
- مدیریت متمرکز چند سرور با همگام‌سازی خودکار (reconciler).

## نصب

نصب از طریق نصب‌کنندهٔ آنلاین انجام می‌شود که **۳ نسخهٔ آخر** منتشرشده را به‌صورت خودکار نمایش می‌دهد و اجازه می‌دهد یکی را برای نصب انتخاب کنید.

### پیش‌نیازها
- سرور لینوکس (Ubuntu / Debian توصیه می‌شود)، معماری `amd64`
- دسترسی `root`

### نصب پنل (control-plane)

روی سرور به‌صورت root این دستور را اجرا کنید:

</div>

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/xpanel-cp/x-net/main/install/xnet.sh)
```

<div dir="rtl">

نصب‌کننده ۳ نسخهٔ آخر را در یک منو نشان می‌دهد؛ شمارهٔ نسخهٔ موردنظر را انتخاب کنید (پیش‌فرض = جدیدترین). سپس بستهٔ همان نسخه دانلود و نصب می‌شود.

نمونهٔ خروجی انتخاب نسخه:

</div>

```text
Available X-NET releases (latest 3):
  1) v1.3.0  —  X-NET v1.3.0 (latest)
  2) v1.2.1  —  X-NET v1.2.1
  3) v1.2.0  —  X-NET v1.2.0

Choose a version to install [1-3, default 1]:
```

<div dir="rtl">

نصب مستقیم یک نسخهٔ مشخص بدون منو:

</div>

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/xpanel-cp/x-net/main/install/xnet.sh) v1.2.3
```

<div dir="rtl">

در نصب اولیه، نصب‌کننده پورت پنل و اطلاعات ادمین را می‌پرسد، سرویس systemd می‌سازد، هستهٔ sing-box را نصب می‌کند و پورت را در فایروال باز می‌کند. در اجرای مجدد (upgrade)، فایل `.env` و دیتابیس حفظ می‌شوند.

> **نصب آفلاین/دستی:** اگر سرور به گیت‌هاب دسترسی ندارد، می‌توانید بستهٔ Release را دستی منتقل کرده و `sudo bash install.sh` را اجرا کنید. جزئیات در انتهای همین فایل.

### نصب نود (agent / data-plane)

همان دستور نصب را روی سرور نود اجرا کنید و در پرسش نقش، مقدار `agent` را انتخاب کنید. سپس در پنل وارد «سرورها و نودها → Register Node» شوید؛ کلیدها به‌صورت خودکار در پنل تولید می‌شوند. این مقادیر را در `/opt/xnet/.env` سرور نود قرار دهید و سرویس را restart کنید:

```bash
# روی سرور agent
nano /opt/xnet/.env
#   NODE_ROLE=agent
#   NODE_API_KEY=xnetnode_...
#   NODE_SECRET_KEY=...
systemctl restart xnet
```

### مدیریت سرویس پس از نصب

</div>

```bash
systemctl status xnet      # وضعیت سرویس
journalctl -u xnet -f      # لاگ زنده
systemctl restart xnet     # اجرای دوباره
```

<div dir="rtl">

> **امنیت:** آدرس `http://IP:PORT` رمزنگاری‌شده نیست. برای محیط واقعی پنل را پشت دامنه + HTTPS (Reverse Proxy مانند Caddy/Nginx) قرار دهید. دیتابیس در `/opt/xnet/data/xnet.db` نگه‌داری می‌شود و با اجرای دوبارهٔ نصب از بین نمی‌رود.

## امکانات کامل

### مدیریت سرویس و کاربران
- **مدیریت اینباندها (Sing-box):** ساخت، ویرایش، کلون، فعال/غیرفعال‌سازی، و استقرار روی چند نود.
- **سیزده پروتکل هسته:** VLESS، VMess، Trojan، Shadowsocks، SOCKS، HTTP، TUIC، Hysteria2، WireGuard، Mixed، TUN، ShadowTLS، NaiveProxy.
- **ترابری‌ها (Transports):** TCP، WebSocket، gRPC، HTTP/2، HTTPUpgrade، QUIC.
- **امنیت/TLS:** Plain، TLS، **Reality**.
- **اکانت‌های SSH:** ساخت کاربر سیستمی Unix، پروتکل‌های SSH-over-WS / TLS / SlowDNS / Dropbear / BadVPN-UDPGW، محدودسازی ترافیک و تعداد ورود همزمان.
- **مدیریت اشتراک‌ها:** صدور لینک اشتراک، QR Code، سقف حجم ترافیک، تاریخ انقضا، سقف دستگاه همزمان، تمدید، و ریست ترافیک.

### معماری چندنودی (Multi-Node)
- نقش‌های **panel** و **agent** در زمان نصب.
- ثبت نود با تولید خودکار کلید API و کلید HMAC در پنل.
- **همگام‌سازی خودکار (reconciler):** استقرار اینباند/اکانت روی نودها و نمایش وضعیت همگام‌سازی per-node.
- **تک‌نود فعال (Single-Active-Node / Follow-Me):** هر کاربر فقط روی یک نود فعال باشد و از طریق لینک اشتراک یا پنل بین نودها جابجا شود.
- انتخاب نودهای مجاز برای هر اینباند و host به‌ازای هر نود.

### امنیت و احراز هویت
- ورود با **JWT** و **احراز هویت دومرحله‌ای (TOTP / Google Authenticator)**.
- نقش‌های دسترسی: **Super Admin**، **Operator / Support**، **Reseller / Agent** (با محدودیت تعداد اشتراک و نودهای مجاز).
- رمزنگاری پسورد با **bcrypt**.
- امضای **HMAC** برای ارتباط امن پنل ↔ نود.

### شبکه و زیرساخت
- مدیریت **دامنه‌ها** با بررسی واقعی وضعیت DNS و SSL.
- صدور و تمدید **گواهی TLS** (Let's Encrypt / ACME) و کلیدهای سفارشی.
- **قوانین مسیریابی (Routing):** outbound detour، GeoSite/GeoIP، AdBlock-DNS، با ترتیب اولویت و تستر زندهٔ بسته.
- پیکربندی **DNS** و کنترل تراکم **TCP BBR**.

### مانیتورینگ و عملیات
- **داشبورد** و **تحلیل ترافیک (Analytics):** مصرف پهنای باند در زمان، توزیع به‌تفکیک پروتکل، و رتبه‌بندی پرمصرف‌ترین کاربران.
- **لاگ‌های سیستم** زنده.
- **پشتیبان‌گیری و بازیابی** دیتابیس (snapshot SQL) و VACUUM.
- **API Management:** توکن Bearer، playground تعاملی، مستندات و SDK نمونه، و فایروال whitelist آی‌پی.

### تجربهٔ کاربری
- رابط **چندزبانه**: فارسی، انگلیسی، روسی، چینی — با فونت اختصاصی هر زبان و جهت RTL/LTR.
- تم روشن/تاریک.
- **بروزرسانی خودکار** صفحات با بازهٔ قابل تنظیم.
- صفحهٔ اشتراک چندزبانه با امکان جابجایی نود توسط کاربر.

## محتویات بستهٔ نصب

این فولدر آرتیفکت‌های آمادهٔ اجراست (بدون سورس):

| فایل/فولدر | توضیح |
|------------|-------|
| `xnet.sh` | نصب‌کنندهٔ آنلاین. ۳ نسخهٔ آخر گیت‌هاب را نشان می‌دهد، بسته را دانلود و `install.sh` را اجرا می‌کند. |
| `xnet-server` | باینری بک‌اند لینوکس (amd64، بدون CGO). API + WebSocket + سرو فرانت روی یک پورت. |
| `dist/` | فرانت‌اند بیلدشده. |
| `install.sh` | نصب‌کنندهٔ تعاملی سرور (به Go/Node نیاز ندارد). |
| `xnet-ssh-apply` | اعمال تنظیمات SSH روی سیستم (آپدیت sshd + کنترل سرویس‌ها). |
| `xnet-install-sshd.sh` | نصب معماری multi-sshd (نمونه‌های WS/TLS/DNS). |
| `xnet-cert-install.sh` | نصب و تنظیم stunnel / گواهی TLS. |
| `healthcheck.sh` | تست سلامت پنل و اشتراک‌ها (بعد از نصب). |

### نصب آفلاین (دستی)

اگر به‌جای نصب آنلاین می‌خواهید بسته را دستی منتقل کنید:

</div>

```bash
# انتقال کل فولدر به سرور
scp -r install root@SERVER_IP:/root/xnet-install

# روی سرور
ssh root@SERVER_IP
cd /root/xnet-install
sudo bash install.sh
```
